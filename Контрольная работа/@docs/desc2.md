# Решение задачи с двумя указателями

## Исходные данные
- Адрес массива a: `0x6C65EFFB58`
- Адрес массива b: `0x6C65EFFB88`

## Предположения
- Массивы типа `int` (4 байта на элемент)
- Массивы содержат: `a[6] = {10, 11, 12, 13, 14, 15}` и `b[6] = {10, 11, 12, 13, 14, 15}`
- Указатели `pa` и `pb` типа `int*`

---

## 1. Адреса и значения элементов массивов

Размер элемента `int` = 4 байта (0x04 в шестнадцатеричном виде).

**Массив a:**
```
&a[0] = 0x6C65EFFB58     a[0] = 10
&a[1] = 0x6C65EFFB5C     a[1] = 11  (FB58 + 4 = FB5C)
&a[2] = 0x6C65EFFB60     a[2] = 12  (FB5C + 4 = FB60)
&a[3] = 0x6C65EFFB64     a[3] = 13  (FB60 + 4 = FB64)
&a[4] = 0x6C65EFFB68     a[4] = 14  (FB64 + 4 = FB68)
&a[5] = 0x6C65EFFB6C     a[5] = 15  (FB68 + 4 = FB6C)
```

**Массив b:**
```
&b[0] = 0x6C65EFFB88     b[0] = 10
&b[1] = 0x6C65EFFB8C     b[1] = 11  (FB88 + 4 = FB8C)
&b[2] = 0x6C65EFFB90     b[2] = 12  (FB8C + 4 = FB90)
&b[3] = 0x6C65EFFB94     b[3] = 13  (FB90 + 4 = FB94)
&b[4] = 0x6C65EFFB98     b[4] = 14  (FB94 + 4 = FB98)
&b[5] = 0x6C65EFFB9C     b[5] = 15  (FB98 + 4 = FB9C)
```

---

## 2. Инициализация указателей

```c
int *pa = a;        // pa указывает на начало массива a
int *pb = &b[5];    // pb указывает на последний элемент массива b
```

**Начальные значения указателей:**
```
pa = 0x6C65EFFB58  (указывает на a[0])
pb = 0x6C65EFFB9C  (указывает на b[5])
```

**Значения по указателям:**
```
*pa = 10  (значение a[0])
*pb = 15  (значение b[5])
```

---

## 3. Выполнение цикла while (*pa < *pb)

Цикл выполняется, пока значение по указателю `pa` меньше значения по указателю `pb`.

### Итерация 1

**Перед проверкой условия:**
```
pa = 0x6C65EFFB58  →  *pa = 10
pb = 0x6C65EFFB9C  →  *pb = 15
```

**Проверка условия:** `10 < 15` → **TRUE**, цикл выполняется

**Вывод в программе:**
```
*pa = 10        *pb = 15        pa = 0x6C65EFFB58        pb = 0x6C65EFFB9C
```

**После выполнения операций:**
```c
pa++;  // pa теперь указывает на a[1]
pb--;  // pb теперь указывает на b[4]
```

**Новые значения указателей:**
```
pa = 0x6C65EFFB5C  (указывает на a[1])
pb = 0x6C65EFFB98  (указывает на b[4])
```

**Почему адреса изменились именно так?**

- `pa++` увеличивает адрес на `sizeof(int)` = 4 байта
  - `0x6C65EFFB58 + 4 = 0x6C65EFFB5C`
- `pb--` уменьшает адрес на `sizeof(int)` = 4 байта
  - `0x6C65EFFB9C - 4 = 0x6C65EFFB98`

### Итерация 4 (цикл завершается)

**Перед проверкой условия:**
```
pa = 0x6C65EFFB64  →  *pa = 13
pb = 0x6C65EFFB90  →  *pb = 12
```

**Проверка условия:** `13 < 12` → **FALSE**, цикл **НЕ выполняется**

Цикл завершается, указатели остаются на текущих позициях.

**Почему цикл остановился?**

Условие `*pa < *pb` стало ложным:
- `*pa = 13` (значение a[3])
- `*pb = 12` (значение b[2])
- `13 < 12` → **FALSE**

## Ключевые моменты

1. **Два указателя движутся навстречу:**
   - `pa` увеличивается (движется вперёд по массиву a)
   - `pb` уменьшается (движется назад по массиву b)

2. **Арифметика указателей:**
   - `pa++` увеличивает адрес на 4 байта (размер int)
   - `pb--` уменьшает адрес на 4 байта
   - Арифметика указателей учитывает размер типа данных

3. **Условие цикла:**
   - Цикл продолжается, пока значение по `pa` меньше значения по `pb`
   - Когда `*pa >= *pb`, цикл завершается

4. **Результат:**
   - После 3 итераций указатели сходятся
   - `pa` указывает на a[3] (значение 13)
   - `pb` указывает на b[2] (значение 12)
   - Условие `13 < 12` ложно, цикл завершается

5. **Важно понимать:**
   - Указатели изменяются, но значения в массивах остаются неизменными
   - Сравниваются значения по указателям, а не сами указатели
   - Постфиксные операции `++` и `--` изменяют указатель после использования текущего значения
