# ✅ **Зачем нужно второе задание**

## **1. Понять разницу между двумя массивами одинаковых значений**

В задании есть:

```c
int a[] = {10,11,12,13,14,15};
int b[] = {10,11,12,13,14,15};
```

Хотя массивы одинаковые — они находятся **в разных местах памяти**.
Это важно для понимания адресов и работы указателей.

---

## **2. Научиться работать с двумя указателями, движущимися навстречу**

```c
int *pa = a;
int *pb = &b[5];
```

* `pa` — начинается с **начала** массива `a`
* `pb` — начинается с **конца** массива `b`

Дальше в цикле:

```c
while (*pa < *pb) {
    pa++;
    pb--;
}
```

Ты видишь, как один указатель идёт вперёд, а другой назад.
Это классическая техника:

* сравнения массивов,
* двухуказательную обработку,
* алгоритмы поиска и сортировки.

---

## **3. Понять разницу между *адресом* и *значением по адресу***

Вывод в цикле показывает:

* адрес `pa`
* значение `*pa`
* адрес `pb`
* значение `*pb`

Это учит чётко различать:

* **указатель**
* **то, куда он указывает**
* **как меняется указатель при инкременте**

---

## **4. Осознать, что арифметика указателей зависит от типа**

Когда мы пишем:

```c
pa++;
```

указатель увеличивается **на sizeof(int)** байт
(например, на 4 байта), а не на 1.

То есть ты понимаешь **почему адреса увеличиваются именно так**.

---

## **5. Научиться отслеживать логику программы через память**

Каждая итерация показывает:

* как указатели сходятся
* когда цикл должен остановиться
* как меняются значения при движении

Это важный навык: понимать, *что происходит в памяти* прямо по программе.

---

## **6. Это база для понимания:**

* работы со строками в С,
* алгоритмов,
* динамической памяти,
* низкоуровневого программирования,
* безопасного доступа к данным,
* указателей в структурах и массивах структур.
