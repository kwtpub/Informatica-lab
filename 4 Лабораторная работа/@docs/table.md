## Выводы по результатам

### Анализ эффективности алгоритмов:

1. **Сортировка вставкой:**
   - **Лучший случай (отсортированный массив):** O(n) - минимальное количество сравнений
   - **Худший случай (обратный массив):** O(n²) - максимальное количество сравнений и перестановок
   - **Средний случай (рандомный массив):** O(n²) - эффективнее пузырьковой и выбором по количеству сравнений

2. **Сортировка пузырьком:**
   - **Лучший случай (отсортированный массив):** O(n²) - все равно выполняет все сравнения
   - **Худший случай (обратный массив):** O(n²) - максимальное количество операций
   - **Средний случай (рандомный массив):** O(n²) - одинаковое количество сравнений с сортировкой выбором

3. **Сортировка выбором:**
   - **Лучший случай (отсортированный массив):** O(n²) - всегда выполняет все сравнения
   - **Худший случай (обратный массив):** O(n²) - минимальное количество перестановок (n-1)
   - **Средний случай (рандомный массив):** O(n²) - минимальное количество перестановок среди всех алгоритмов

### Сравнительный анализ:

- **По количеству сравнений:** Сортировка вставкой показывает лучшие результаты для отсортированных и рандомных массивов
- **По количеству перестановок:** Сортировка выбором требует наименьшее количество перестановок
- **Для отсортированных массивов:** Сортировка вставкой значительно эффективнее (O(n) против O(n²))
- **Для обратных массивов:** Все алгоритмы показывают одинаковое количество сравнений, но сортировка выбором требует меньше перестановок
